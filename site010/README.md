# MusicLoop (site010)

## 개요
- **사이트 이름:** MusicLoop
- **사이트 ID:** site010
- **포트 번호:** 9339
- **기술 스택:** Node.js, Express, Vanilla HTML/CSS/JS

## 실행 방법
```bash
cd site010
npm install
npm start
```
서버가 실행되면 브라우저에서 `http://localhost:9339` 으로 접속할 수 있습니다.

## API 엔드포인트 목록
- `GET /api/health` - 서버 상태 확인
- `GET /api/songs` - 전체 곡 혹은 타입별 곡 조회 (popular 필터 포함)
- `GET /api/playlists` - 공개된 플레이리스트 및 내 플레이리스트 조회
- `GET /api/playlists/:id` - 특정 플레이리스트 상세 조회
- `GET /api/playlists/:id/songs` - 특정 플레이리스트에 속한 곡 목록 조회
- `GET /api/recent` - 최근 재생한 곡 목록 조회
- `GET /api/artists` - 추천 아티스트 목록 조회

## 정상 작동 기능 목록 (인터랙션)
1. **네비게이션 탭 전환 (홈 / 장르탐색 / 라이브러리)**
2. **플레이리스트 클릭 시 상세 모달 오픈 및 곡 목록 렌더링**
3. **인기 곡, 최근 재생 곡 클릭 시 하단 플레이어 컨트롤 바 연동 (재생 시뮬레이션)**
4. **플레이어 하트(좋아요) 토글 기능**
5. **장르(전체/일렉트로닉/신스팝/락) 해시태그 필터링을 통한 탐색**
6. **곡 제목 및 아티스트 이름 실시간 검색 필터링**
7. **하단 플레이어 재생/일시정지 상태 토글**
8. **추천 알고리즘 새로고침 액션**
9. **사이드바 유저 상태 및 토스트 알림 연동**
10. **개발자 테스트용 버그 트리거 토글/버튼 조작**

## 의도된 오류 (3개)
이 프로젝트는 강화학습 모델(PPO) 훈련을 위해 다음 3개의 오류를 포함하고 있습니다.
1. **site010-bug01 (DB 오류):** DB 관계 매핑 오류 (플레이리스트 곡 조회 시 엉뚱한 최근 재생 곡이 불러와짐)
2. **site010-bug02 (네트워크 오류):** 부분 응답 데이터 손상 (인기 곡 API에서 필수 필드가 삭제되어 반환됨)
3. **site010-bug03 (보안 오류):** 비공개 리소스 접근 제어 실패 (비공개 플레이리스트를 권한 없이 조회 가능)
